# 小白看这里

这是给 Windows 用户看的 Galcode 新手手册。`main` 分支现在是全平台通用版，但这份文档会继续按 Windows 小白路线来讲。

本项目采取MIT协议分发，使用时不必通知我，各位玩的开心，做出更有意思的二创就好。

这个项目不需要任何的编程基础，小白也能轻松驾驭。相比于社区以往的自动化项目，这个项目打通了从灵感到出片的全套工作流，让你喜欢的二创作者或者说你能够以惊人的速度快速产出。

先说人话：Galcode 可以把你的一句中文脑洞，变成一个 WebGAL 小剧场，再帮你录成 `final.mp4`。你不用先学 WebGAL，也不用先成为命令行高手。

## 你需要准备什么

请用 Windows 10 或 Windows 11。下面命令建议在 PowerShell 里运行。

你需要三个东西：

1. Git：用来下载项目。
2. Node.js：用来运行 Galcode。
3. FFmpeg：用来录制视频。

最省事的安装方式是用 `winget`：

```powershell
winget install Git.Git
winget install OpenJS.NodeJS.LTS
winget install Gyan.FFmpeg
```

装完以后，关闭 PowerShell，再重新打开一个新的 PowerShell。然后检查：

```powershell
git --version
node --version
npm --version
ffmpeg -version
```

看到版本号就行。Node.js 需要 20 或更高版本。

## 下载 Galcode

找一个你放项目的地方，比如桌面或 `D:\Projects`，然后运行：

```powershell
git clone https://github.com/Huanuyn1/Galcode.git
cd Galcode
```

默认的 `main` 分支就是通用版，Windows 用户直接用它就行。

以后所有命令都在这个 `Galcode` 文件夹里运行，不要在别的地方敲命令。

## 第一次安装

如果你下载的是“带双击启动器”的压缩包，优先找这个文件：

```text
Galcode-windows.exe
```

直接双击它。它会自动准备 Node.js、安装依赖并打开 Galcode 图形界面。第一次运行需要联网，窗口里跑很多字是正常的，不是坏了，是机器在努力把舞台搭起来。

如果你手里只有这一个 `Galcode-windows.exe` 文件，也可以直接双击。它会自己下载 Galcode 项目文件到电脑里，再继续安装依赖并打开 GUI。你不需要先学会 Git，也不需要先打开 PowerShell。唯一要记住的是：第一次运行一定要联网。

如果你是从 GitHub 克隆源码，或者压缩包里没有 `Galcode-windows.exe`，再用下面的方法。

在 `Galcode` 文件夹里运行：

```powershell
.\install.bat
```

这个过程会安装 Electron、Playwright、WebGAL 等依赖，可能需要等一会。终端看起来安静的时候不要急着关，让它自己跑完。

安装完后做一个检查：

```powershell
.\galcode --help
```

能看到帮助说明，就说明 Galcode 已经能启动了。

如果你想最省事，也可以直接运行：

```powershell
.\start.bat
```

它会自动安装依赖，然后进入 Galcode 聊天模式。第一次跑的时候慢一点很正常，像灯第一次写歌词，安静不是卡住，是在憋大的。

## 配置 AI

想让 Galcode 按你的脑洞写剧情，需要配置 API key：

```powershell
.\galcode configure
```

它会问你三件事：

```text
OpenAI 兼容接口地址
模型
API key
```

如果你不知道前两个填什么，请打开ai供应商的官方文档。

也可以手动编辑 `.env`：

```powershell
copy .env.example .env
notepad .env
```

里面大概长这样：

```text
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4.1-mini
OPENAI_API_KEY=你的_key_写这里
```

API key 是你的私人物品，不要截图发群里！！据说祥爹当年就是这么欠下169亿的……

## 正式生成一个作品

最推荐新手使用这一条：

```powershell
.\galcode make --theme "灯和爱音在排练前和解" --record --duration 60 --out outputs\tomori-anon-rain
```

看成片：

```powershell
start .\outputs\tomori-anon-rain\final.mp4
```

你可以把主题换成自己的，比如：

```powershell
.\galcode make --theme "爽世不小心说出真心话，大家都假装没听见" --record --duration 60 --out outputs\soyo-truth
```

```powershell
.\galcode make --theme "乐奈觉得今天的排练很有趣" --record --duration 45 --out outputs\rana-interesting
```

```powershell
.\galcode make --theme "祥子路过排练室门口，但没有进去" --record --duration 60 --out outputs\sakiko-door
```

## 进入聊天模式

如果你想和 Galcode 慢慢聊创作方向：

```powershell
.\galcode
```

进去以后可以直接输入中文：

```text
我想写一个雨夜短篇，灯和爱音因为歌词理解不同吵了一下，但最后没有彻底说开
```

也可以用这些命令：

| 命令 | 用途 |
| --- | --- |
| `/brainstorm 主题` | 先要几个点子，不急着生成 |
| `/make 主题` | 按这个主题生成作品 |
| `/yolo` | 不讨论了，直接来 |
| `/config` | 重新配置 API key / 模型 / 接口地址 |
| `/quit` | 退出 |

如果你脑子里只有一句“爱希牛逼”，也可以直接说。它会继续问你角色、口味和时长。

## 为什么命令前面有 `.\`

PowerShell 默认不会直接运行当前文件夹里的程序，所以要写：

```powershell
.\galcode
```

如果你在 CMD 里，也可以直接写：

```cmd
galcode
```

如果你真的想在任何目录都能打 `galcode`，可以之后再研究 PATH 或 `npm link`。新手阶段不用管这个，先在项目目录里用 `.\galcode` 就好。

如果 `.\galcode` 也提示找不到，说明你不在 `Galcode` 项目目录里。

## 常用参数不用背

| 参数 | 人话解释 |
| --- | --- |
| `--offline` | 不连 AI，跑本地 demo |
| `--record` | 生成后录成视频 |
| `--duration 60` | 目标时长 60 秒 |
| `--out outputs\name` | 指定输出目录 |
| `--theme "..."` | 告诉 AI 你想写什么 |
| `--capture electron` | 推荐录制方式，后台录制 |
| `--capture screenshot` | 备用录制方式，适合排错 |

记不住就记这一条：

```powershell
.\galcode make --theme "你的主题" --record --duration 60
```

## 成品在哪里

默认会放在 `outputs` 文件夹里。比如你指定：

```powershell
--out outputs\tomori-anon-rain
```

那么目录大概长这样：

```text
outputs\tomori-anon-rain\
├── story.json
├── asset-manifest.json
├── game\
│   ├── scene\start.txt
│   ├── figure\live2d\
│   └── template\
└── final.mp4
```

看成片：打开 `final.mp4`。

想看 AI 写了什么：打开 `story.json`。

想看 WebGAL 脚本：打开 `game\scene\start.txt`。

## 常见报错

### `node` 不是内部或外部命令

说明 Node.js 没装好，或者 PATH 没刷新。

先重新打开 PowerShell。如果还不行，重新安装 Node.js LTS，安装时确认勾选 Add to PATH。

### `npm` 不是内部或外部命令

和上面一样，通常是 Node.js 没装好。重新打开 PowerShell，或者重装 Node.js。

### `ffmpeg` 不是内部或外部命令

运行：

```powershell
winget install Gyan.FFmpeg
```

装完以后重新打开 PowerShell，再检查：

```powershell
ffmpeg -version
```

### 提示缺少 API key

只是体检项目就加 `--offline`：

```powershell
.\galcode yolo --offline --record --duration 30
```

想正式创作就配置：

```powershell
.\galcode configure
```

### `npm install` 很慢或失败

先重试一次：

```powershell
.\install.bat
```

如果还是不行，可能是网络问题。换网络、开代理，或者稍后再试。

### 录制失败

先确认不录制也能生成：

```powershell
.\galcode yolo --offline --duration 30 --out outputs\no-record-test
```

再单独录制：

```powershell
.\galcode record outputs\no-record-test --capture electron --duration 30 --video-out outputs\no-record-test\final.mp4
```

如果 Electron 或依赖看起来坏了，重新装依赖：

```powershell
.\install.bat
```

录制这块偶尔会抽风，多数时候是依赖没装齐。

### 页面空白或没有 Live2D

先跑最小体检：

```powershell
.\galcode yolo --offline --record --duration 30 --out outputs\live2d-test
```

如果还是空白，复制终端里的完整报错，提交issue，或者在能找到本人的地方甩过来。

## 写主题的小技巧

不要只写：

```text
写一个 MyGO 故事（你让llm怎么想……）
```

可以多给一点方向：

```text
灯和爱音，雨夜，排练前误会，最后不要大和解，只要愿意继续说话（这是gpt5.5喜欢的，奥尔特曼你坏事做尽）
```

```text
爽世和睦，表面平静，实际都知道对方没有说真话，结尾留一点刺（不是gpt怎么就喜欢这种剧情啊喂）
```

```text
乐奈觉得今天的排练很有趣，但大家完全不知道她为什么这么说
```

好主题一般包含四样东西：

1. 谁登场。
2. 在哪里。
3. 有什么小冲突。
4. 结尾要甜一点、痛一点，还是怪一点。
